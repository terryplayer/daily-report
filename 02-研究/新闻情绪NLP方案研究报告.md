# 📰 新闻情绪NLP · API方案研究报告

---

## 一、需求分析

| 需求 | 说明 |
|:----|:-----|
| 数据频率 | 每日一次（盘后或盘前） |
| 覆盖范围 | A股主流财经新闻 + 持仓个股相关 |

## 三、情感分析方案

### 方案1：HuggingFace 本地模型（推荐 ✅）

| 项目 | 说明 |
|:----|:-----|
| 模型 | `uer/roberta-base-finetuned-chinanews-chinese` |
| 方式 | 本地加载，`transformers` 库推理 |
| 费用 | **零成本**（仅CPU/GPU计算） |
| 准确率 | 中文新闻情感分类约85% |
| 依赖 | `pip install transformers torch` |
| 速度 | CPU批处理 100条/秒 |

**代码示例（零成本实现）：**
```python
from transformers import pipeline
classifier = pipeline(
    "sentiment-analysis",
    model="uer/roberta-base-finetuned-chinanews-chinese"
)
result = classifier("半导体板块今日大涨，龙头股集体爆发")
# → [{'label': 'POSITIVE', 'score': 0.985}]
```

## 四、推荐方案

```
数据源: Tushare News API（已有点数，零成本）
情感模型: HuggingFace uer/roberta（本地推理，零成本）
汇总方式: 按板块/个股聚合
频率: 每日盘后
总成本: ¥0/月
```

**技术栈：**
- `tushare.pro.news()` → 获取新闻
- `transformers` + 中文预训练模型 → 情感打分
- Python dict聚合 → 板块级情绪指标
- JSON输出 → 供 `gen_premarket_report.py` 调用

**预计代码量：** ~200行 Python（独立模块 `calc_news_sentiment.py`）

## 六、风险与注意事项

1. **Tushare新闻可用性** — 需要确认 `pro.news()` 的实时性和覆盖度（Tushare Pro积分版可用）
2. **模型泛化能力** — 财经领域情感与通用情感有差异，可能需要少量标注数据微调
3. **新闻时效性** — 盘后采集可能漏掉盘中突发新闻
4. **板块映射** — 新闻中提到的公司名→持仓代码映射需要维护
