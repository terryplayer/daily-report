## 一、框架

**四个认知维度（十三哥提出）：**

| | 视角 | 核心假设 | 判断依据 | 时效 |
|:-:|:-----|:---------|:---------|:----:|
| **① 统计** | 数学科学 | 历史会重演 | 数据规律 | 短~中 |
| **② 时序/系统** | 动态演化 | 过程可建模 | 状态追踪 | 中短~中 |
| **③ 经济/金融** | 货币价值 | 价值锚存在 | 基本面估值 | 中~长 |
| **④ 行为/情绪** | 人性博弈 | 情绪驱动波动 | 群体心理 | 超短~短 |

```
         ┌──────────────┐
         │  交易决策     │
         ├──────┬───────┤
         │ ④行为│ ①统计 │
         │(人想)│(规律) │
         ├──────┼───────┤
         │ ②时序│ ③经济 │
         │(演化)│(价值) │
         └──────┴───────┘
```

> 四个维度不互斥，互补形成决策支撑金字塔。

| 子类 | 已有 | 可补充 |
|:-----|:----:|:------|
| 描述统计 | ✅ 均值/方差 | — |

| 降维聚类 | ✅ PCA、K-means | 层次聚类 |

**已上线：**

| 模型 | 帮什么决策 | 权重 |
|:-----|:----------|:----:|

## 🏗️ 报告设计理念

> 以下内容说明进阶版HTML模板的设计逻辑，即四象限模型演进完毕后，四个报告（盘前/午间/收盘/周复盘）的组织方式。

### 核心理念：从「展示数据」转向「辅助决策」

现有模板的设计逻辑是**「我们有这些数据，给你看看」**——拆成RS表、动量表、多因子表，每个模型一块，数据摆在那你自己判断。

进阶模板的设计逻辑是**「你今天要做这些决策，这些信息能帮你」**——板块按决策问题组织，不是按模型组织。

```
现有盘前：外围市场 → 大盘预判 → 模型信号 → 今日关注 → 持仓观察 → 策略
进阶盘前：四象限总览(我今天整体看多/看空？)
          → 外围+情绪(外部环境+情绪怎么样？)
          → 大盘趋势(技术面支持做多吗？)
          → 板块四维评分(哪个板块各方面都强？)
          → 个股四维评分(哪只股票各方面都好？)
          → 关键信号(今天有什么明确的交易信号？)
          → 配对策略(有没有套利机会？)
          → 综合策略(所以我到底该怎么操作？)
```

### 设计逻辑一：报告间递进闭环

四个报告不是孤立的，而是一个**「预测→验证→总结→进化→再预测」**的闭环：

```
盘前 ──→ 午间 ──→ 收盘 ──→ 周复盘
预测      验证      总结      进化
 │         │         │         │
 └── 预测 ─┘    ↑     └── 全周数据 ──┘
                  │
            每天沉淀 → RAG快照索引 → 下次盘前用
```

| 报告 | 要回答的核心问题 | 对应新增板块 |
|:-----|:----------------|:-----------|
| **盘前** | 今天该不该交易？该交易什么？ | 四象限总览→信号聚合→配对 |
| **午间** | 上午走偏了没？要调整吗？ | 趋势验证→多维异动→午后策略 |
| **收盘** | 今天什么有效？明天怎么看？ | Fama-French+美林+GARCH+四维准确率 |
| **周复盘** | 这周赚/亏在哪？下周怎么做？ | K-means+配对+模型进化复盘 |

### 设计逻辑二：每个报告有「核心问题」

四个报告各承担一个角色，**逐层递进**，不重复展示同样的数据：

- **盘前：预测与决策** — 集中所有可在盘前获取的信息，给出方向判断
- **午间：验证与修正** — 比较盘前预测vs上半日实际，判断是否要调整
- **收盘：总结与归因** — 全天回顾，分析什么对了什么错了，为明日做准备
- **周复盘：进化与调整** — 周度模型准确率评估，指导权重微调

### 设计逻辑三：「⏳演进中」是建筑设计图，不是缺失

模板中大量出现的`⏳演进中`是故意的——**这些模板是为四象限完全演进完毕后的状态设计的**，而不是为今天的数据做的。

- **架构先行** — 不等到模型都上线了再想怎么展示
- **渐进填充** — 每个演进任务上线，对应的模板板块自然就有数据了
- **反向驱动** — 模板告诉生成脚本「我需要什么数据」，而不是脚本「我可以生成什么」

### 从「数据陈列」到「决策矩阵」的转变

```
现有模式：
  数据 → 模型 → 表格 → 你判断

进阶模式：
  你今天要做什么决策？
     ↓
  四个象限给你四个视角
     ↓
  每个视角有1~2个关键指标
     ↓
  融合给出建议
     ↓
  你只需要确认或否决
```

现有模板是**工具箱**，告诉你有什么工具。

进阶模板是**施工图纸**，告诉你先做什么后做什么、哪里该用什么工具。
