# 🏗️ 选股评分系统重构方案 v1.1

> 状态：方案设计 | 更新：2026-06-09 | 维护：小十三 🌀

---

## 一、现状问题

```
当前架构：
  daily_score_gen.py ─┬─ 东方财富/Tushare 数据采集
                       ├─ RS/多因子/多因素评分计算
                       ├─ Excel 选股评分输出
                       └─ 市场全景 HTML 生成（2.4MB，数据内嵌）
                                  ↓
  一个脚本全包 → 一处挂全部挂 → 数据混在一起 → 出问题难排查
```

**今天踩的坑：**
1. 盘中跑脚本，Tushare 没数据 → 东财兜底
2. 东财数据字段格式不同 → 换手率/量比/成交额变天文数字
3. Excel 被覆盖 → 6/8 数据也被搞乱
4. 修复数据 → 又搞丢了市场全景的持仓按钮
5. 市场全景每次生成 2.4MB HTML，数据内嵌不可单独修改

**根因：** 采集、计算、展示 三层没有隔离，全在一锅里。

## 四、市场全景网页改造详情

### 现状（重构前）

```
selestock/index.html = 2.4MB
├── CSS (~50行)
├── HTML 骨架 (~30行)
├── JS 渲染引擎 (~80行)
└── data = [...5000+条股票数据]  ← 占 99% 体积，写死在HTML里

缺点：
- 数据变了必须重新生成整个 HTML
- 无法单独修改某只股票的数据
- 2.4MB 加载慢
- 部署要整个文件推送到 GitHub
```

### 目标（重构后）

```
selestock/index.html = ~8KB
├── CSS (~50行)
├── HTML 骨架 (~30行)
├── JS 渲染引擎 (~150行)
└── data → fetch('/data/today') 从 API 实时加载

优点：
- 数据实时，无需重新生成 HTML
- 可以手动修数据后页面立刻反映
- 8KB 秒加载
- 一次部署管永久
```

## 八、公网部署方案

### 8.1 问题

市场全景网页部署在 Cloudflare Pages（远程），data_service.py 运行在本地 mac（:18790）。
远程浏览器访问 Cloudflare 页面时无法连接 `localhost:18790`。

### 8.2 方案对比

| 方案 | 难度 | 费用 | 延迟 | 安全 |
|:-----|:----:|:----:|:----:|:----:|
| A. Cloudflare Tunnel | ⭐⭐ | 免费 | 低 | ✅ TLS+鉴权 |
| B. 阿里云 ECS + Nginx | ⭐⭐⭐ | ¥50/月 | 中 | ✅ 防火墙 |
| C. ngrok | ⭐ | 免费版限速 | 高 | ⚠️ 公开URL |
| D. 内网穿透(FRP) | ⭐⭐⭐⭐ | 免费 | 低 | ⚠️ 需自建 |

**推荐：方案 A — Cloudflare Tunnel（最省心，零成本）**

---

### 8.3 Cloudflare Tunnel 方案详解

#### 原理

```
浏览器(用户)                    Cloudflare 边缘节点             本地 Mac
    │                               │                          │
    ├─ https://daily-report-3ai     │                          │
    │    .pages.dev/selestock/ ───► │                          │
    │                               ├── Cloudflare Tunnel ──► │
    │                               │    (wss加密隧道)         │ localhost:18790
    │    ◄── 响应数据 ────────────── │                          │
    │        (通过CF边缘缓存)        │                          │
```

Cloudflare Tunnel 创建一个加密隧道从 Cloudflare 边缘节点直达你本地 Mac 的 :18790 端口。
用户浏览器请求 Cloudflare Pages 上的页面 → 页面 JS 请求 API → Cloudflare 通过隧道转发到本地 → 本地响应。
整个过程不需要公网 IP，不需要开放端口，TLS 加密。

#### 实施步骤

##### 第一步：安装 cloudflared

```bash
# macOS
brew install cloudflared

# 验证
cloudflared --version
```

##### 第二步：登录 Cloudflare 账号

```bash
cloudflared tunnel login
# 浏览器打开链接 → 选择 daily-report-3ai.pages.dev 所在域名 → 授权
```

##### 第三步：创建隧道

```bash
# 创建
cloudflared tunnel create daily-report-api

# 会生成一个 credentials JSON 文件
# ~/.cloudflared/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.json
```

##### 第四步：配置隧道

创建 `~/.cloudflared/config.yml`：

```yaml
tunnel: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
credentials-file: /Users/shisan/.cloudflared/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.json

originRequest:
  connectTimeout: 10s
  noTLSVerify: true

# 将 api.your-domain.com 指向本地 18790
#（需要先在 Cloudflare DNS 加一条 CNAME 记录）

ingress:
  - hostname: api.daily-report.example.com
    service: http://localhost:18790
  - service: http_status:404
```

##### 第五步：DNS 配置

在 Cloudflare DNS 面板加一条：

| 类型 | 名称 | 目标 |
|:----|:-----|:-----|:----|
| CNAME | `api` | `xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.cfargotunnel.com` |

##### 第六步：启动隧道（开机自启）

```bash
# 安装为系统服务
sudo cloudflared service install

# 或手动后台运行
nohup cloudflared tunnel run daily-report-api > /tmp/tunnel.log 2>&1 &
```

##### 第七步：修改前端 API 地址

Cloudflare Pages 上的市场全景页面 API URL 改为公网地址：

```javascript
// selestock/index.html
// 本地：const API = ''（同源）
// 公网：const API = 'https://api.daily-report.example.com'
```

可以在部署时动态判断域名来切换：

```javascript
const API = location.hostname === '127.0.0.1' || location.hostname === 'localhost'
  ? ''
  : 'https://api.daily-report.example.com';
```

| 条件 | 说明 | 状态 |

### 8.5 替代方案（如果没自有域名）

#### 方案 C1: localhost + 浏览器直连

当前状态。你本地打开 `http://127.0.0.1:18790/` 直接使用。
适合你自己一个人用，不需要远程访问。

#### 方案 C2: 内嵌数据版本并存

同时保留两个市场全景页面：
- `selestock/index.html` — 动态版（本地用，从 localhost API 加载）
- `selestock/offline.html` — 内嵌数据版（2.4MB，部署到 CF Pages，无法交互）

#### 方案 C3: GitHub + Actions 定时重建

把 data_service 的 API 响应定时提交到 GitHub，
CF Pages 构建时把数据内嵌回 HTML。
不需要公网 API，但有 ~1小时延迟。
