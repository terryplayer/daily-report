## 二、后端的实时同步机制

### 2.1 FastAPI 轻量后端（端口 18790）

```python
# scripts/watchlist_api.py
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
import json, os

WORKSPACE = os.path.dirname(os.path.dirname(__file__))
app = FastAPI()
app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"])

@app.get("/watchlist")
def get_watchlist():
    """获取当前持仓清单（市场全景页面JS启动时调用）"""
    with open(os.path.join(WORKSPACE, "config", "stocks.json")) as f:
        data = json.load(f)
    return {"codes": [s["code"] for s in data["watchlist"]]}

@app.post("/watchlist/add")
def add_stock(code: str, name: str):
    """加仓"""
    path = os.path.join(WORKSPACE, "config", "stocks.json")
    with open(path) as f:
        data = json.load(f)
    if any(s["code"] == code for s in data["watchlist"]):
        return {"status": "exists"}
    # 板块自动归类
    sector = "其他"
    if code.startswith("688"): sector = "科技/半导体"
    elif code.startswith("300"): sector = "创业板/科技"
    elif code.startswith("60"): sector = "主板"
    elif code.startswith(("000","002")): sector = "深市主板"
    data["watchlist"].append({"code": code, "name": name, "sector": sector})
    data["version"] = "2026-06-07"
    with open(path, 'w') as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
    _sync_template(data)  # 立即同步
    return {"status": "ok", "total": len(data["watchlist"])}

@app.delete("/watchlist/remove")
def remove_stock(code: str):
    """减仓"""
    path = os.path.join(WORKSPACE, "config", "stocks.json")
    with open(path) as f:
        data = json.load(f)
    data["watchlist"] = [s for s in data["watchlist"] if s["code"] != code]
    data["version"] = "2026-06-07"
    with open(path, 'w') as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
    _sync_template(data)
    return {"status": "ok", "total": len(data["watchlist"])}

def _sync_template(data):
    """实时同步到 template-stocks.json（标准版报告用）"""
    tpl_path = os.path.join(WORKSPACE, "scripts", "template-stocks.json")
    with open(tpl_path, 'w') as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
```

### 2.2 双向同步机制

```
你点「加仓」
    │
    ├─④ JS 更新页面状态（即时反馈）
    │
    └─① POST → FastAPI
               │
               ├─② 更新 config/stocks.json ✅
               │
               └─③ _sync_template() → 全量覆盖 template-stocks.json ✅
                                           │
                     ┌──────────────────────┘
                     ▼
              下次 09:00 stock_analysis.py 运行
                     │
                     ├─ 从 config/stocks.json 加载（已包含新股票）
                     ├─ 同步到 template（已是最新，无变化）
                     └─ 重新生成 cache
                           │
                           ├─ 盘前简报 ✅
                           ├─ 午间监测 ✅
                           ├─ 收盘简报 ✅
                           ├─ 进阶版报告 ✅
                           └─ 所有演进模型 ✅
```

**关键时间线：**

```
加仓操作（任意时间）
  │
  ├── 0.1秒内 → config/stocks.json 更新 ✅
  ├── 0.1秒内 → template-stocks.json 同步 ✅
  ├── 0.1秒内 → 市场全景页面状态刷新 ✅（JS实时）
  │
  ├── 下一次 09:00 → cache 刷新，盘前简报包含新股票 ✅
  ├── 下一次 12:00 → 午间监测包含 ✅
  ├── 下一次 15:30 → 收盘简报包含 ✅
  └── 下一次 21:00 → 复盘包含 ✅
```

## 三、方案优化点

### 相比原始方案的优化

| 项 | 原始方案 | 优化版 |
|:--|:--------|:------|
| 交互反馈 | 提交后等后端操作 | JS 即时更新状态 + 按钮loading态 |
| 数据刷新 | 重新生成整个 HTML | 前端 JS 动态获取持仓清单，不刷新页面 |
| 板块归类 | 手动填 | 根据代码前缀**自动推断**，少一个输入 |
| 加仓去重 | 无校验 | API 层去重，重复加返回 exists |
| 错误处理 | 无 | 按钮 loading、失败提示 |
| 端口隔离 | 无 | 18790 独立端口，不与网关冲突 |

### 关于市场全景 HTML 的更新

持仓操作**不触发市场全景 HTML 重新生成**（因为不需要——持仓状态是前端 JS 动态渲染的）。

市场全景 HTML 仍然在每日 20:00 选股评分任务中正常更新（更新股票涨跌数据），持仓状态列由 JS 实时读取 FastAPI 接口来显示。

两者互不干扰：
- **持仓状态**：JS 实时获取 ✅
- **涨跌数据**：每日 20:00 刷新 ✅
