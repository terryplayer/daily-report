# 🗂️ 持仓股票清单统一方案（v2.0）

## 一、现状与问题

### 三副本现状

```
config/stocks.json                  ← 持仓配置源（77只，JSON数组）
scripts/template-stocks.json        ← 标准版报告用（77只，JSON数组，需手动同步）
scripts/stock_analysis.py           ← 分析引擎用（77只，Python硬编码列表，需手动同步）
   第22~110行: WATCHLIST_FULL = [...]  ← 硬编码
   第112行:    WATCHLIST_CODES = [...] ← 从WATCHLIST_FULL推导
```

### 各自下游

```
config/stocks.json
  └─ gen_daily_snapshot.py（每日快照）

template-stocks.json
  ├─ gen_premarket_report.py（盘前简报）
  ├─ gen_midday_report.py（午间监测）
  └─ gen_closing_report.py（收盘简报）

stock_analysis.py:WATCHLIST_FULL（硬编码，77只）
  └─ 生成 /tmp/stock_analysis_cache.json
       ├─ gen_advanced_reports.py（进阶版报告）
       ├─ rf_injector.py（随机森林模型）
       ├─ hermes_review.py（Hermes复盘）
       └─ 下游所有演进模型
          ├─ 协整配对（天孚↔新易盛等）
          ├─ Hurst指数
          ├─ 卡尔曼滤波
          ├─ ARIMA/GARCH
          ├─ STL分解
          ├─ K-means聚类
          ├─ PCA降维
          ├─ 马尔可夫链
          └─ Ridge回归...
```

### 核心问题

往 config/stocks.json 加一只新股票 → 必须同时手动改另外 2 个文件（共 3 处），否则部分报告不感知。

## 二、改动方案

### 2.1 核心改动：只改一个文件

**`scripts/stock_analysis.py`** 第22~110行

```python
# ─── 删除这 90 行硬编码 ───
WATCHLIST_FULL = [
    {"code": "688549", "name": "中巨芯-U", "sector": "科技/半导体"},
    ...  # 77只全部硬编码
]
WATCHLIST_CODES = [s["code"] for s in WATCHLIST_FULL]  # 第112行也删除

# ─── 替换为动态加载 ───
import json

def load_watchlist():
    """从统一配置源 config/stocks.json 加载持仓股票清单"""
    path = os.path.join(WORKSPACE, "config", "stocks.json")
    try:
        with open(path) as f:
            data = json.load(f)
        watchlist = data.get("watchlist", [])
        if not watchlist:
            print(f"⚠️ 警告: {path} 中 watchlist 为空")
        return watchlist
    except FileNotFoundError:
        print(f"❌ 错误: 配置文件 {path} 不存在")
        return []
    except json.JSONDecodeError as e:
        print(f"❌ 错误: 配置文件格式错误 {e}")
        return []

WATCHLIST_FULL = load_watchlist()
WATCHLIST_CODES = [s["code"] for s in WATCHLIST_FULL]
```

**改动量**：删除约 90 行硬编码 API 列表，新增约 20 行动态加载代码。

**效果**：WATCHLIST_FULL 的值现在取决于 config/stocks.json 的内容，下游所有引用 WATCHLIST_FULL 和 WATCHLIST_CODES 的地方（共 14 处）自动生效，零改动。

### 2.2 同步机制

**方案 A：stock_analysis.py 启动时同步（推荐）**

在 `load_watchlist()` 成功读取后，自动将 config/stocks.json 同步到 `template-stocks.json`：

```python
def load_watchlist():
    path = os.path.join(WORKSPACE, "config", "stocks.json")
    try:
        with open(path) as f:
            data = json.load(f)
        watchlist = data.get("watchlist", [])
        
        # ─── 自动同步到 template-stocks.json ───
        template_path = os.path.join(WORKSPACE, "scripts", "template-stocks.json")
        try:
            with open(template_path, 'w') as tf:
                json.dump(data, tf, ensure_ascii=False, indent=2)
        except Exception as e:
            print(f"⚠️ 同步到 template-stocks.json 失败: {e}")
        
        return watchlist
    except Exception as e:
        print(f"❌ 加载持仓清单失败: {e}")
        return []
```

**为什么选方案 A 不是 B：**
- 方案 B（软链）在 macOS + Git 环境下会有跨平台兼容问题
- 方案 A 在 `stock_analysis.py` 每次运行时自动同步一次，0 维护成本
- stock_analysis.py 每天 09:00 盘前采集和 15:30 收盘采集都会运行，确保 template-stocks.json 始终是最新的

### 板块自动归类

从市场全景加股票时，用户不一定知道该归到哪个板块。后端可以**根据股票代码自动推断**：

| 代码前缀 | 所属板块 |
|:--------|:--------|
| 60xxxx | 主板 |
| 000xxx / 002xxx | 深市主板/中小板 |
| 300xxx | 创业板 |
| 688xxx | 科创板 |
| 8xxxxx | 北交所 |

如果无法自动归类，默认标记为「其他」，用户可随时在 config/stocks.json 中手动调整。

### FastAPI 后端示例

```python
# ~/openclaw/workspace/scripts/watchlist_api.py
from fastapi import FastAPI
import json, os, subprocess

app = FastAPI()
WORKSPACE = os.path.dirname(os.path.dirname(__file__))

def get_sector_by_code(code):
    """根据股票代码自动推断板块"""
    if code.startswith("60"): return "主板"
    if code.startswith(("000", "002")): return "深市主板"
    if code.startswith("300"): return "创业板/科技"
    if code.startswith("688"): return "科技/半导体"
    return "其他"

@app.post("/watchlist/add")
def add_stock(code: str, name: str):
    """添加股票到持仓清单"""
    path = os.path.join(WORKSPACE, "config", "stocks.json")
    with open(path) as f:
        data = json.load(f)
    
    # 去重检查
    if any(s["code"] == code for s in data["watchlist"]):
        return {"status": "exists", "message": "已在持仓中"}
    
    # 追加
    sector = get_sector_by_code(code)
    data["watchlist"].append({
        "code": code, "name": name, "sector": sector
    })
    data["version"] = datetime.now().strftime("%Y-%m-%d")
    
    with open(path, 'w') as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
    
    # 触发同步 + 缓存刷新
    subprocess.Popen(["python3", os.path.join(WORKSPACE, "scripts", "stock_analysis.py"), "--update", "--json"],
                     cwd=WORKSPACE)
    
    return {"status": "ok", "message": f"{name} 已加入持仓", "total": len(data["watchlist"])}
```

## 四、改动风险与回退

### 风险评估

| 风险 | 概率 | 应对措施 |
|:----|:---:|:---------|
| config/stocks.json 格式异常 | 低 | try/except 兜底，失败时打印错误而非崩溃；保留空列表作为回退值 |
| 文件不存在 | 极低 | FileNotFoundError 捕获，打印清晰提示 |
| 同步写入时磁盘满/权限不足 | 极低 | 捕获写入异常，仅打印警告，不中断主流程 |
| 新加的股票代码在 Tushare 查不到 | 低 | stock_analysis.py 的遍历逻辑中会跳过查不到的股票，不会崩溃 |
| 并发写（罕见） | 极低 | 同步只在启动时读+写一次，不涉及并发写逻辑 |

### 回退方案

```bash
# 1. 如果新代码有问题，一键恢复 git 版本
cd /Users/shisan/.openclaw/workspace
git checkout -- scripts/stock_analysis.py

# 2. 或者更精细的：手动取消注释旧代码、注释新代码
#    第22行附近有清晰的标记
```

如果 `stock_analysis.py` 改动后出问题，git checkout 1 秒回退，不影响任何已有数据。
