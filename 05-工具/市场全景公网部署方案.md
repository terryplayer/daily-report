# 🌐 市场全景 · 公网部署方案

> 解决公网访问时增减持仓失败的问题（`Failed to fetch`）
> 当前问题：Cloudflare Pages 公网页面 → JS 调 `127.0.0.1:18790` → 连接拒绝

```
💻 本地 Mac
   └── watchlist_api.py :18790 ✅ 仅本机
```

## 二、公网部署方案

### 方案A：轻量部署（不改代码，10分钟）

把 `watchlist_api.py` 直接丢到公网服务器：

```bash
# 服务器上
scp watchlist_api.py root@your-server:/opt/
python3 watchlist_api.py &
```

改两处：
1. **`watchlist_api.py`** 监听地址 `127.0.0.1` → `0.0.0.0`
2. **前端 JS** API 地址从 `127.0.0.1:18790` 改为 `https://your-server.com:18790`

### 方案B：FastAPI + Nginx 反向代理（推荐）

既然十三哥喜欢 FastAPI，可以重写为专业方案：

```
┌─ 公网服务器 ───────────────────────────────────┐
│                                                  │
│  Nginx（443 HTTPS）                              │
│    ├─ /api/watchlist  → FastAPI :8000           │
│    ├─ /selestock/     → 静态文件（可选）         │
│    └─ /               → Cloudflare Pages 代理    │
│                                                  │
│  FastAPI (uvicorn, :8000)                        │
│    ├─ GET    /api/watchlist                      │
│    ├─ POST   /api/watchlist/add                  │
│    └─ DELETE /api/watchlist/remove               │
│                                                  │
│  watchlist_api.py → 服务端运行                   │
│  config/stocks.json → 服务端同步                 │
│                                                  │
└──────────────────────────────────────────────────┘
```

**Nginx 关键配置：**
```nginx
server {
    listen 443 ssl;
    server_name your-server.com;
    
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 三、有公网服务器后的优化方向

### 优化1：动态加载，告别 2.4MB

当前：5515只股票内联 JSON → 2.4MB HTML
优化后：HTML 仅框架（5KB），数据 API 分页获取

### 优化2：实时数据，告别日更

当前：每天 20:00 跑选股评分，数据固定一天
优化后：打开页面时从 API 实时获取最新行情（盘中可用）

### 优化3：完整持仓管理界面

当前：简陋加仓/减仓按钮
优化后：
- 搜索添加（输入代码/名称搜索 A 股）
- 批量操作
- 自定义分组
- 导入/导出

### 优化4：后台自动更新

当前：手动跑 `daily_score_gen.py`
优化后：服务器 cron 定时更新，自动推送到 Cloudflare Pages
