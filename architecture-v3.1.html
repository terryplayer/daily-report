3.2 卡尔曼替换 MRD — 详细设计

✅ 设计决策：只换算法，不换接口。MRD 在统一模型中的权重/位置/字段含义全部不变，只把底层从「(close-MA)/MA」换成「(当前价-卡尔曼平滑价)/卡尔曼平滑价」。

3.2.1 替换前后对比

对比项旧 MRD新 卡尔曼偏离
计算方式mrd = (close - MA10) / MA10 × 100kf_dev = (close - kalman_price) / kalman_price × 100
偏离基准10日均价（固定窗口）卡尔曼估计价（自适应噪声 + 全历史）
趋势感知❌ 无✅ 状态向量包含趋势分量
波动率❌ 无✅ 输出 volatility / support / resistance
自适应❌ 固定窗口✅ process/observe noise 按个股波动自适应
活跃股误判❌ 高换手率→高MRD误判✅ 与换手率无关，纯价格统计
超买阈值>+8%>+8%（不变）
超卖阈值<-8%<-8%（不变）

# 4. Cloudflare Pages 部署
#    API token 从环境变量读取（服务器和本地都支持）

    CLOUDFLARE_API_TOKEN=$(python3 -c "
import json, os
token = os.environ.get('CLOUDFLARE_API_TOKEN', '')
if not token:
    # 无需手动配置token（deploy_cloudflare.py自动读取）
    try:
        with open(cfg) as f:
            token = json.load(f)[0]['api_token']
    except: pass
print(token)
" 2>/dev/null)
fi

export CLOUDFLARE_API_TOKEN

if [ -n "$CLOUDFLARE_API_TOKEN" ]; then
    python3 $WORKSPACE/scripts/deploy_cloudflare.py 2>&1

    echo "⚠️  CLOUDFLARE_API_TOKEN 未设置，跳过部署"

10.4 关键设计决策

✅ 决策1: 飞书推送 — 服务器直推

理由：服务器是主力运行环境，直接调用飞书webhook最快、最可靠。
不需要本地转发：避免了「本地离线→飞书收不到」的单点故障。
测试：本地开发时可用独立测试webhook，不影响生产推送。
✅ 决策2: Hermes复盘 — 服务器主力

理由：复盘紧跟数据生成（18:00，收盘后），在服务器上跑最自然。
本地可重跑：复盘JSON同步到本地后，本地Hermes可加载历史JSON做深度交互复盘。
不依赖本地在线：复盘不等待本地，独立完成。
✅ 决策3: 服务器运行方式 — cron + 脚本

推荐：cron + shell/Python脚本（最简方案）
· 不需要FastAPI（没有外部API调用需求）
· cron直接调度 daily-cycle.md 各section
· 可选：一个小型健康检查端点（python3 -m http.server 8080 提供 /health）
不推荐FastAPI：增加复杂度但无收益（报告是静态HTML，部署走CF Pages）
✅ 决策4: 路径和API Keys — 环境变量统一

服务器：全部走环境变量，不依赖文件路径（/tmp除外）
本地：保持现有文件配置（token文件、config目录），向后兼容
gen_common.py：env优先 + file fallback，一套代码双环境运行

    "$SERVER_DATA/data/stock_history.json" \

def calc_kalman_deviation(code, history):
    from utils.kalman_filter import get_kalman_levels
    closes = [d['close'] for d in history.get(code,{}).get('days',[])]
    if len(closes) < 5:
        return {'kalman_dev': 0, 'smoothed_price': closes[-1] if closes else 0,
                'trend': 0, 'support': 0, 'resistance': 0, 'signal': '数据不足'}
    levels = get_kalman_levels(closes)
    cp = closes[-1]
    sp = levels['price']
    dev = (cp - sp) / sp * 100
    return {
        'kalman_dev': round(dev, 2),
        'smoothed_price': round(sp, 2),
        'trend': round(levels['trend'], 4),
        'support': round(levels['support'], 2),
        'resistance': round(levels['resistance'], 2),

                  '偏高·关注压力' if dev > 4 else (
