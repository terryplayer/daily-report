# 🎒 中学生AI学习系统 — 产品规格书

> 版本：v0.1 | 日期：2026.05.31 | 状态：规划中

---

## 一、产品概述

### 1.1 产品定位
面向中学生的**全科AI学习助手**，覆盖「学 → 练 → 考 → 评 → 讲」完整闭环。学生可以学习知识点、生成模拟试卷、在线作答、AI自动批改并逐题讲解。

### 1.2 核心能力
| 能力 | 说明 |
|------|------|
| **知识学习** | 按学科/年级/章节浏览知识点，AI生成图文讲解 |
| **智能出卷** | 根据知识点、难度、题型自动生成模拟试卷 |
| **在线考试** | 浏览器内作答，支持选择题/填空题/简答题/计算题 |
| **自动批改** | AI批改主观题，客观题自动判对错 |
| **逐题讲解** | 每道题生成详细解析，含解题思路、知识点回顾、易错点 |
| **学习报告** | 阶段性学习数据统计，薄弱点分析，提分建议 |

### 1.3 目标用户
- **学生**（初一至高三）：核心用户，使用学习+考试功能
- **家长**（可选）：查看学习报告
- **老师**（可选）：查看班级学情（V2）

### 2.2 详细功能描述

#### 2.2.1 知识学习模块

**知识点结构**
```
学科 → 年级 → 教材版本 → 章节 → 知识点
```

每个知识点包含：
- **知识讲解**：AI生成的图文说明，适合中学生理解
- **公式/定理**：数学、物理、化学公式展示（LaTeX渲染）
- **例题示范**：1-2道典型例题+解析
- **关联知识点**：前置知识和后续知识链接
- **常见易错点**：学生容易犯的错误提示

**交互方式**
- 浏览器Web页面浏览
- AI Agent对话式学习（"我不懂二次函数的顶点公式，能给我讲讲吗？"）
- 学习进度追踪

#### 2.2.2 AI智能出卷

**出卷参数**
| 参数 | 说明 | 可选值 |
|------|------|--------|
| 学科 | 出卷学科 | 语文/数学/... |
| 年级 | 目标年级 | 初一~高三 |
| 知识点 | 覆盖哪些知识点 | 多选 |
| 难度 | 整体难度 | 基础/中等/困难 |
| 题型 | 包含题型 | 单选/多选/填空/判断/简答/计算 |
| 题量 | 总题数 | 5~50 |
| 时长 | 建议时长 | 10~120分钟 |

**出卷流程**
1. 用户设定参数
2. OpenClaw Agent 调用 AI 生成试卷
3. 试卷结构化存储（JSON格式，含题目、选项、答案、分值）
4. 前端渲染为可作答的试卷页面

#### 2.2.3 批改与讲解

**批改能力**
| 题型 | 批改方式 | 准确度 |
|------|---------|--------|
| 单选题 | 自动比对 | 100% |
| 多选题 | 自动比对 | 100% |
| 判断题 | 自动比对 | 100% |
| 填空题 | AI语义匹配 | ~95% |
| 简答题 | AI评分+评语 | ~85% |
| 计算题 | AI步骤评分 | ~80% |

**讲解结构**
每道题的讲解包含：
```
┌─ 题目（原题展示）
├─ 📌 正确答案
├─ ✅ 解题思路（分步骤）
├─ 📖 知识点回顾（相关知识点链接）
├─ ⚠️ 常见错误（如果是错的，分析错误原因）
└─ 💡 举一反三（同类题型推荐/变式练习）
```

### 2.3 OpenClaw AI Agent 设计

#### Agent 角色定义
| Agent | 职责 | 触发方式 |
|-------|------|---------|
| **学习助教** | 讲解知识点、回答学习疑问 | 对话触发 |
| **出卷师** | 根据参数生成试卷 | 页面表单触发 |
| **阅卷官** | 批改主观题、生成评分 | 交卷后自动触发 |
| **讲题师** | 逐题生成讲解内容 | 批改完成后触发 |
| **学习分析师** | 生成学习报告、提分建议 | 数据分析触发 |

#### Agent 交互流程

**出卷流程（示例）**
```
用户提交出卷参数
    ↓
Web后端调用 OpenClaw Agent「出卷师」
    ↓
Agent 调用 LLM 生成题目
    ↓
Agent 结构化输出（JSON）
    ↓
写入数据库
    ↓
前端展示试卷
```

**批改流程（示例）**
```
学生提交答卷
    ↓
Web后端接收
    ↓
客观题自动批改（本地比对）
    ↓
主观题→调用 OpenClaw Agent「阅卷官」
    ↓
Agent 逐题评分+写评语
    ↓
批改完成→触发「讲题师」Agent
    ↓
逐题生成讲解
    ↓
存入数据库→前端展示
```

### 3.2 技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| **前端框架** | Next.js 14+ (App Router) | SSR/SSG，考试页面需客户端交互 |
| **UI组件** | shadcn/ui + Tailwind CSS | 美观、快速开发 |

| **部署** | Vercel + Cloudflare | Web在Vercel，Agent在本地/云 |
| **公式渲染** | KaTeX | 数学公式渲染 |

### 3.3 数据库模型（核心表）

```prisma
model Subject {
  id        String   @id @default(cuid())
  name      String   // 语文、数学...
  grade     String   // 初一~高三
  chapters  Chapter[]
  createdAt DateTime @default(now())
}

model Chapter {
  id          String       @id @default(cuid())
  subjectId   String
  subject     Subject      @relation(fields: [subjectId], references: [id])
  name        String       // 章节名
  order       Int          // 排序
  knowledgePoints KnowledgePoint[]
}

model KnowledgePoint {
  id        String      @id @default(cuid())
  chapterId String
  chapter   Chapter     @relation(fields: [chapterId], references: [id])
  name      String      // 知识点名
  content   Json?       // AI生成的讲解内容（结构化）
  createdAt DateTime    @default(now())
  updatedAt DateTime    @updatedAt
}

model Exam {
  id          String    @id @default(cuid())
  userId      String
  title       String    // 试卷标题
  subject     String
  grade       String
  difficulty  String    // easy/medium/hard
  questions   Json      // 题目数组
  answers     Json?     // 学生答案
  score       Int?      // 得分
  totalScore  Int       // 总分
  status      String    // draft/pending/graded
  timeLimit   Int?      // 分钟
  createdAt   DateTime  @default(now())
  submittedAt DateTime?
  gradedAt    DateTime?
}

model QuestionReview {
  id          String   @id @default(cuid())
  examId      String
  exam        Exam     @relation(fields: [examId], references: [id])
  questionIdx Int
  studentAnswer Json?
  correctAnswer Json?
  isCorrect   Boolean?
  score       Int?
  explanation Json?    // AI生成的讲解
  createdAt   DateTime @default(now())
}
```

## 六、关键设计决策

### 6.1 为什么用 OpenClaw Agent 而不是直接调 LLM API？
- Agent 有**工具调用能力**（查知识库、写数据库、调用外部API）
- 出卷/批改是**有状态的多步骤流程**，Agent 天然适合
- 后期可以接入**多个 Agent 协作**（出卷师出完题→审题师检查质量）
- OpenClaw 的 cron/heartbeat 可以用来做**定时任务**（如每日一题推送）

### 6.2 出卷质量如何保证？
- 每个知识点关联了**难度系数**和**典型题目模板**
- Agent 出卷时参考模板保证题型规范
- 人工审核机制（V2）：老师可以审核并调整AI出的试卷
- 反馈闭环：评分低的题目标记，下次出卷优化

### 6.3 主观题批改准确度如何？
- 简答题采用 AI 语义评分 + 关键词匹配
- 计算题分步骤评分，每步给出得分点
- 学生可以对批改结果**申诉**，触发重新批改
- 初期建议搭配人工抽检

## 七、风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| AI出卷题目质量不稳定 | 学习体验差 | 模板约束+人工审核+反馈迭代 |
| 主观题批改偏差 | 学生不信任 | 显示评分细则+申诉机制 |
| LLM API 延迟 | 出卷/批改慢 | 流式输出+加载状态优化+超时重试 |
| 数学公式渲染复杂 | 显示异常 | 统一用KaTeX+MD格式+测试覆盖 |
| 数据库性能（大量试卷） | 页面加载慢 | 分页+缓存+索引优化 |
