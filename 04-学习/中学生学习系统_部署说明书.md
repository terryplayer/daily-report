# 🚀 中学生AI学习系统 — 部署说明书

> 版本：v0.1 | 适用系统：macOS / Linux | 部署方式：Vercel + 本地OpenClaw

---

## 一、环境要求

### 1.1 本地开发环境

| 工具 | 版本要求 | 用途 |
|------|---------|------|
| Node.js | >= 22.x | 运行Next.js开发服务器 |
| pnpm | >= 9.x | 包管理（推荐） |
| PostgreSQL | >= 15.x | 数据库（本地或Supabase） |

| 服务 | 用途 | 费用参考 |
|------|------|---------|
| **Vercel** (vercel.com) | 部署 Next.js Web应用 | 免费版够用 |

## 二、本地开发部署

### 2.1 项目初始化

```bash
# 1. 创建 Next.js 项目
npx create-next-app@latest study-ai --typescript --tailwind --app --src-dir
cd study-ai

# 2. 安装依赖
pnpm add prisma @prisma/client next-auth shadcn-ui
pnpm add @radix-ui/react-dialog @radix-ui/react-dropdown-menu lucide-react
pnpm add katex react-markdown remark-math rehype-katex
pnpm add zustand @tanstack/react-query

# 3. 初始化 shadcn/ui
pnpm dlx shadcn-ui@latest init

# 4. 初始化 Prisma
pnpm dlx prisma init --datasource-provider postgresql
```

### 2.2 数据库配置

```bash
# 环境变量 (.env.local)
DATABASE_URL="postgresql://user:password@localhost:5432/study_ai"
# 或使用 Neon（推荐）
# DATABASE_URL="postgresql://user:password@ep-xxx.us-east-2.aws.neon.tech/study_ai?sslmode=require"
```

```bash
# 创建数据库迁移
pnpm dlx prisma db push
# 或生成迁移文件
pnpm dlx prisma migrate dev --name init

# 生成 Prisma Client
pnpm dlx prisma generate
```

### 2.3 初始化种子数据

创建种子脚本 `prisma/seed.ts`，填充学科/章节/知识点基础数据：

```bash
pnpm dlx prisma db seed
```

种子数据包含：
- 9个学科的基础结构
- 各年级的章节划分
- 初始知识点模板

### 2.4 启动开发服务器

```bash
pnpm dev
# 访问 http://localhost:3000
```

## 三、OpenClaw Agent 配置

### 3.1 安装 OpenClaw

```bash
# macOS
curl -fsSL https://download.openclaw.ai/install.sh | sh

# 启动 Gateway
openclaw start
```

### 3.2 创建 Agent 配置文件

在项目根目录创建 `agents/` 文件夹，每个 Agent 一个配置文件：

```yaml
# agents/exam-maker.yaml
name: 出卷师
description: 根据参数生成中学生考试试卷
model: deepseek/deepseek-v4-flash
prompt: |
  你是一位经验丰富的中学科目出卷老师。
  根据以下参数生成试卷：
  - 学科：{subject}
  - 年级：{grade}
  - 知识点：{knowledge_points}
  - 难度：{difficulty}
  - 题型：{question_types}
  - 题量：{count}

  要求：
  1. 题目难度分布合理（基础:中等:困难 = 5:3:2）
  2. 题目表述清晰，符合中学生认知水平
  3. 答案准确，解析详细
  4. 输出严格按 JSON 格式

tools:
  - db_query  # 查询知识库获取知识点内容
  - web_search  # 搜索最新题型参考
```

### 3.3 Agent 调用方式

**方案 A：通过 OpenClaw Gateway API 调用（推荐）**

```typescript
// lib/agent.ts
const GATEWAY_URL = process.env.OPENCLAW_GATEWAY_URL || 'http://localhost:9999';

export async function generateExam(params: ExamParams) {
  const res = await fetch(`${GATEWAY_URL}/api/agents/exam-maker/run`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      input: params,
      mode: 'agent',
    }),
  });
  return res.json();
}
```

**方案 B：通过 OpenClaw SDK 调用（开发环境）**

```typescript
import { OpenClaw } from 'openclaw-sdk';

const claw = new OpenClaw({ gateway: 'http://localhost:9999' });
const result = await claw.agent('exam-maker').run({ /* 参数 */ });
```

### 3.4 Agent 注册到 Gateway

```bash
openclaw agent register agents/exam-maker.yaml
openclaw agent register agents/grader.yaml
openclaw agent register agents/tutor.yaml
```

## 四、生产环境部署

### 4.1 Vercel 部署

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录
vercel login

# 3. 部署
vercel --prod
```

### 4.2 环境变量配置

在 Vercel Dashboard 设置以下环境变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `DATABASE_URL` | 数据库连接串 | `postgresql://...` |
| `OPENCLAW_GATEWAY_URL` | OpenClaw Gateway地址 | `http://your-server:9999` |
| `OPENCLAW_API_KEY` | OpenClaw认证密钥 | `sk-xxx` |
| `LLM_API_KEY` | DeepSeek/OpenAI密钥 | `sk-xxx` |
| `NEXTAUTH_SECRET` | 认证密钥 | `openssl rand -base64 32` |
| `NEXTAUTH_URL` | 回调地址 | `https://study.yourdomain.com` |

### 4.3 OpenClaw Gateway 生产部署

如果 Gateway 需要部署到服务器：

```bash
# 方案一：同一台 VPS 上运行
openclaw start --port 9999

# 方案二：Docker 部署（推荐）
docker run -d \
  --name openclaw-gateway \
  -p 9999:9999 \
  -v $(pwd)/agents:/agents \
  -e OPENCLAW_API_KEY=sk-xxx \
  ghcr.io/openclaw/gateway:latest
```

### 4.4 Nginx 反向代理（可选）

```nginx
server {
    listen 443 ssl;
    server_name api.study.yourdomain.com;

    # 转发到 OpenClaw Gateway
    location / {
        proxy_pass http://127.0.0.1:9999;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 五、AI Agent 实现细节

### 5.1 出卷 Agent 逻辑

```yaml
# agents/exam-maker.yaml - 完整配置
name: 出卷师
model: deepseek/deepseek-v4-flash
thinking: high
timeoutSeconds: 120
prompt: |
  你是{grade}{subject}出卷老师。

  ## 出卷规则
  1. 题目必须覆盖指定知识点
  2. 难度分布：基础40% / 中等40% / 困难20%
  3. 客观题（选择/判断）占60%，主观题（简答/计算）占40%
  4. 每题含：题干、选项(如有)、答案、解析、分值、难度等级

  ## JSON输出格式
  {
    "title": "试卷标题",
    "totalScore": 100,
    "timeLimit": 90,
    "questions": [
      {
        "type": "choice|fill|judge|short_answer|calculation",
        "difficulty": "easy|medium|hard",
        "score": 5,
        "stem": "题目内容，支持LaTeX公式用 $...$ 包裹",
        "options": {"A":"选项A", "B":"选项B"}, // 选择题需要
        "answer": "正确答案",
        "explanation": "详细解析"
      }
    ]
  }

tools:
  - db_query:
      description: 查询指定知识点的内容和例题
      parameters:
        query: string  # SQL查询

  - web_search:
      description: 搜索最近考试真题参考
      parameters:
        query: string
```

### 5.2 批改 Agent 逻辑

```yaml
# agents/grader.yaml
name: 阅卷官
model: deepseek/deepseek-v4-flash
thinking: high
prompt: |
  你是{subject}阅卷老师，严格按评分标准批改。

  批改规则：
  1. 客观题：直接比对答案
  2. 填空题：语义匹配，同义词/近义词视为正确
  3. 简答题：按得分点评分，每题给出评分细则
  4. 计算题：分步骤给分，写出每个步骤的得分

  输出格式：
  {
    "questionIdx": 0,
    "score": 8,
    "maxScore": 10,
    "isCorrect": false,
    "comment": "评语",
    "stepScores": [
      {"step": "第一步：列出公式", "score": 3, "maxScore": 3},
      {"step": "第二步：代入计算", "score": 2, "maxScore": 4}
    ],
    "errorAnalysis": "错误原因分析"
  }
```

### 5.3 讲解 Agent 逻辑

```yaml
# agents/tutor.yaml
name: 讲题师
model: deepseek/deepseek-v4-flash
thinking: high
prompt: |
  你是{subject}老师，用中学生能理解的语言讲解这道题。

  讲解结构：
  1. 📌 **正确答案**：清晰给出
  2. ✅ **解题思路**：分步骤，用中学生能理解的语言
  3. 📖 **知识点回顾**：回顾解题用到的知识点
  4. ⚠️ **错误分析**（如果做错）：分析为什么会错
  5. 💡 **举一反三**：给一个类似的变式题

  要求：
  - 语言亲切但专业
  - 数学公式用LaTeX包裹
  - 多鼓励，少批评
```

## 六、Cursor 辅助开发建议

### 6.1 项目初始化阶段

在 Cursor 中打开项目，用 Composer 模式：

```
Prompt:
用 Next.js 14 (App Router) + shadcn/ui + Tailwind CSS + Prisma + PostgreSQL 
创建一个中学生AI学习系统的项目骨架。

包含：
1. Prisma Schema（用户、学科、章节、知识点、试卷、答题记录）
2. 基础布局（侧边栏导航 + 主内容区）
3. 首页仪表盘（学习概览、最近考试列表）
4. API路由基础结构
```

### 6.2 AI Agent 对接阶段

```
Prompt:
创建 OpenClaw Agent 调用封装层。

3. api/exam/grade/route.ts - 批改API路由

需要有超时处理、错误重试、流式响应支持
```

### 6.3 考试页面开发

```
Prompt:
创建一个在线考试页面 /exam/[id]，使用 shadcn/ui 组件。

功能：
1. 从API加载试卷数据（题目列表）
2. 答题界面：选择题用RadioGroup，填空题用Input，简答题用Textarea
3. 题目导航（上一题/下一题/题号快速跳转）
4. 倒计时显示

```

### 7.4 常见问题

| 问题 | 排查方法 |
|------|---------|
| Agent调用超时 | 检查 LLM API 密钥是否有效，查看 `openclaw logs` |
| 数据库连接失败 | 检查 `DATABASE_URL` 是否正确，网络是否可达 |
| 试卷生成慢 | 考虑使用快速模型（如 DeepSeek Flash），减少题量 |
| 数学公式不渲染 | 检查 KaTeX 是否正确加载，公式格式是否为 `$...$` |
