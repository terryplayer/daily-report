🎯 核心原则
按"可训练 vs 不可训练"拆分。
· 数据采集：原始数据获取，无计算
· 计算分析：固定公式，同输入同输出
· 模型评分：参数可通过历史数据优化
📁 文件独立
· data_collector.py（采集）
· analysis_engine.py（分析）
· calc_model.py（评分）
三个模块各有独立入口、独立输出、独立测试

1. 分类标准

1.1 三层划分

层文件判定规则示例
采集 data_collector.py 从外部API获取原始数据，落地到本地文件。不含计算逻辑 新浪行情fetch / Tushare日线 / 研究员采集 / 宏观数据
分析 analysis_engine.py 参数取值通过经典公式或统计推断确定，同输入同输出 MA/RSI/MACD/布林/KDJ、动量/BBW/卡尔曼、RS排名
评分 calc_model.py 参数取值需通过历史数据回测确定最优值 统一模型权重、RF模型.pkl、阈值映射、评分工具

1.2 边界案例处理

边界项当前归属建议理由
多因子6维权重硬编码在stock_analysis.py→评分可通过历史回测优化
RS分档阈值(A+/A/B/C/D)硬编码→评分等级区间可被回测优化
卡尔曼噪声协方差代码写死留分析当前没问题，未来如需调优再留接口
RS计算窗口(5日)硬编码留分析经典参数
ARIMA(p,d,q)拟合未实现→分析统计推断，非ML训练
DCF估值未实现→分析纯公式，无参数可训练
相对估值未实现拆分百分位计算→分析；阈值映射→评分
配对策略未实现拆分协整检验→分析；z-score阈值→评分

2. 模块设计

2.1 数据采集 → data_collector.py（新建，~400行）

职责：从外部API采集原始数据，落地到 data/ 目录。不含任何计算逻辑。
源采集内容产出文件触发
新浪财经实时行情（93只持仓的close/change_pct/volume）data/stock_history.json09:00/12:00/15:30
Tushare Pro日线数据、daily_basic、涨跌停data/stock_history.json 补充09:00
研究员(researcher.py)北向资金、融资余额、PCR、资金流data/research_cache.json09:00（盘前）
宏观(macro_fetch.py)PMI/CPI/PPI/GDP/社融data/macro_cache.json按需
隔夜外盘美股/美债/美元/A50data/overseas_cache.json09:00（盘前）
输出: data/*.json（原始数据文件，未经任何计算加工）

data/
├── stock_history.json    ← 93只持仓K线（最新收盘价/成交量）
├── research_cache.json   ← 北向/融资/PCR/资金流原始值
├── macro_cache.json      ← 宏观指标原始值
├── oversea_cache.json    ← 隔夜外盘原始值（可选）
└── tushare_token.txt     ← API凭据（不采集）

2.2 计算分析 → analysis_engine.py（新建，~600行）

职责：读取采集层的原始数据，产出量化指标。所有函数确定性计算，同输入同输出。
类别函数/模块来源
技术指标calc_ma / calc_rsi / calc_macd / calc_bollinger / calc_kdj从 stock_analysis.py 搬
get_tech_signals / calc_all_tech_signals搬
calc_momentum / calc_bbw / calc_kalman_deviation搬
RS+波动calc_rs / calc_volatility_alerts / calc_combined_signals搬
板块calc_sector_rotation搬
演进·统计协整 / Hurst / 马尔可夫 / STL / PCA / K-means新模块
演进·宏微观美林时钟 / ARIMA / DCF新模块
演进·原始值估值百分位 / PCR原始值 / 融资变化率新模块（计算层只产出原始值）
输入: data/stock_history.json + data/research_cache.json + data/macro_cache.json

2.3 模型评分 → calc_model.py（新建，~500行）

职责：读取分析层的量化指标 + 已训练好的模型文件，产出综合评分。所有参数可训练优化。
类别函数/模块来源
评分工具normalize_score / score_to_signal / score_to_action从 stock_analysis.py 搬
统一模型calc_unified_model(analysis_output)替代 get_model_summary（权重可训练）
calc_multi_factor_score(analysis_output)搬（权重提到配置层）
calc_historical_similarity(analysis_output)搬（FAISS参数可调）
ML模型rf_predict / ridge_predict搬（读取 models/*.pkl）
演进·映射估值百分位→评分 / PCR→情绪 / 融资→情绪新模块（阈值可训练）
演进·复杂四象限融合评分 / 自适应权重 / 配对z-score新模块
输入: /tmp/analysis_output.json + models/*.pkl
输出: /tmp/scoring_output.json（结构化评分数据）

4. 数据流+接口

  data_collector.py（采集原始数据·不可训练）
        ↓
  data/stock_history.json + research_cache.json + macro_cache.json
        ↓
  analysis_engine.py（计算指标·不可训练）
        ↓
  /tmp/analysis_output.json    ← 结构化指标，schema v1.0
        ↓
  calc_model.py（模型评分·可训练） ← 读取 analysis_output + models/*.pkl
        ↓
  /tmp/scoring_output.json     ← 结构化评分，schema v1.0
        ↓
  gen_report.py --mode xxx     ← 读取两个 output，生成 HTML
        ↓
  deploy.sh → Cloudflare Pages（部署上线）

4.1 接口契约

先定义 schema 再动代码。 三者之间的数据结构必须在拆分前定义好，确保拆分后上下游不受影响。 
第1层接口: data/ → analysis_engine.py
  数据文件格式: stock_history.json / research_cache.json / macro_cache.json
  schema 版本: 当前已有，稳定

第2层接口: analysis_engine.py → calc_model.py  
  中间文件: /tmp/analysis_output.json
  schema 版本: v1.0（需设计）
  说明: 所有消费者不变，新增一个 analysis_output.json 作为评分模块的输入

第3层接口: calc_model.py → gen_report.py
  中间文件: /tmp/scoring_output.json
  schema 版本: v1.0（需设计）
  说明: 原有 /tmp/stock_analysis_cache.json 保留作为兼容层
